## Расчет ресурсов

### База данных

3 копии: 
- CPU: 1 * 3 = 3
- RAM: 4 * 3 = 12

### Кэш

3 копии:
- CPU 1 * 3 = 3
- RAM 4 * 3 = 12

### Фронтенд

5 копий:
- CPU 0.2 * 5 = 1
- RAM 50Mb * 5 = 250Mb

### Бэкенд

10 копий:
- CPU 1 * 10 = 10
- RAM 600Mb * 10 = 6000Mb = 6

### Итого

| Компонент |CPU| RAM |
|-|-|-|
|База данных|3|12|
|Кэш|3|12|
|Фронтенд|1|0.25|
|Бэкенд|10|6|
|**Итого**|**17**|**30.25**|

### Логика размещения

* БД и кэш - stateful-ноды
* Фронт + бэк - stateless-ноды

### Отказоустойчивость
#### stateful-ноды

Штатно на каждой ноде 1 БД и 1 Кэш, при оказе БД и кэш пееезжает на 2 другие:
* первая нода 2 БД + 1 Кеш
* вторая нода 1 БД + 2 Кеш
То есть потребуется запас в RAM - 12 и CPU - 3

Служебные ресурсы:
- CPU - 1
- RAM - 2

Итого на одну ноду потребуется:
- CPU 3 + 1 = 4
- RAM 12 + 2 = 14 ~ 16

#### stateless-ноды бэк+фронт)
- CPU 10 + 1 = 11
- RAM 6 + 0.25 = 6.25

Нужно пережит отказ 1 ноды
Штатно 3 ноды будут загружены примерно:
* Бэкенд 4 + 3 + 3
* Фронтенд 2 + 2 + 1

При падении нагрузка распределится на 2 ноды:
- CPU (10 + 1) / 2 = 5.5
- RAM (6 + 0.25) /2 = 3.125

Служебные ресурсы:
- CPU - 1
- RAM - 2

Итого на одну ноду потребуется:
- CPU 5.5 + 1 = 6.5 ~ 8
- RAM 3.125 + 2 = 5.125 ~ 8
